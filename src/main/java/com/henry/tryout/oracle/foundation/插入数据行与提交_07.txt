我们已经研究了 如何从你的表中 获取数据，
但是如果表是空的，那么 获取数据的手段 就没有用了
是时候 开始添加东西了

插入有两种形式：单行 和 多行
顾名思义，单行插入 每一次将一行数据 添加到你的表中
要做到这一点 你的语句是：
insert into 表名 values(<待插入的数据>)
这提供了你要插入的数据

执行此操作，你需要为 按照表中相同顺序的每一列 提供一个值，
如果以错误的顺序赋值，如果数据类型不兼容的话，你就会收到错误消息
或者 更加令人困惑的是 插入会起作用，但值会出现在错误的位置上

为了避免这种混淆，你可以在表名后 显式指定列，这会使你的代码 更加健壮
并且 更容易检查 你是否犯了错误

如果是只添加几行的话，那么 单行插入非常好用。
但是如果要添加 数百行 甚至 数千行，那么 单行插入 就会花非常多的功夫。

一次添加所有行 就会快得多。
为了做到这一点，请使用 多行插入。
这会添加查询的结果
insert into toys select * from store

这使你能够有效地将数据 从一张表复制到另一张表中。
与单行插入相比，多行插入 还有另一个优势：能够一次向多个表添加数据
有两种方法可以做到这一点：#1 有条件的； #2 无条件的

对于查询返回的每一行，“无条件多表插入” 会在每张表中都插入一行
为此，请以 insert all 来 开始你的SQL
然后添加表，它们的列 和 它们的values子句

像多行插入一样，你需要一个查询语句 来 作为你的数据源；

你还可以使用此技术 将多个单行插入 改为一个语句，
只需要列出values子句中的所有数据；
insert all
    into toys values(xxx)
    into toys values(ooo)

    select * from dual; // 并有一个返回一行的虚拟查询

如果你不希望 查询中的每一行 都进入每张表，那么你需要“带条件的多行插入”
这类似于多行插入，但是在每个表之前，添加一个 带条件的when子句：
只有满足条件的行 才会进入表格。

如果你继续使用insert all，那么 查询中的行 将会出现在 when子句为真的每张表中。
但有时候你想要确保 每一行 只能出现在一张表中，为此 请使用 insert first.

Oracle数据库 在第一个为真的when子句添加一行，从上向下评估。
所以，现在你有了一个充满数据的表格，还有一个重要的步骤：确保你保存了它！
否则，如果出现任何问题，你所做的更改将会丢失

为了防止这种情况，你需要提交~
这会保存你的更改，并且还有另一个重要的影响：
当你向表中添加行时，只有你自己 可以看到你所做的更改，其他访问数据库的人 无法访问它们。
而一旦你发出提交，其他人就可以看到 你所做的更改了。

但是，如果你犯了错误 并且 不想要保存要怎么办呢？
你要怎么撤销它？
输入rollback 这将撤销自上次提交以来的所有更改

== side note ==
大多数数据库客户端 都有一个自动提交的属性，
这个属性表明它 是否会在每个语句之后 或者 关闭连接时 为你保存数据，
确保检查客户端中的设置方式，否则 你最终可能会得到一些不幸的结果
== end side note ==
